# 首页 AI 资讯瀑布流设计

## 概述

在聚合页首页底部新增一个「资讯」模块，展示 AI + 泛科技领域的最新资讯。数据通过 RSSHub 聚合多个来源，经 DeepSeek AI 筛选和总结后展示。

## 需求摘要

- **内容范围：** AI 为主 + 泛科技
- **展示位置：** 首页底部，轻量展示
- **展示数量：** 固定 20 条，不翻页
- **卡片样式：** 标签 + 卡片（分类标签、标题、AI 总结、日期）
- **排列方式：** 列表流（单列横向排列）
- **点击行为：** 展开显示更多摘要 + 原文链接
- **数据更新：** 每 20 分钟更新一次
- **AI 筛选：** DeepSeek 从原始资讯中筛选最有价值的 20 条
- **AI 总结：** 每条资讯由 AI 生成 50 字以内的总结
- **筛选标准：** AI 自主判断

## 数据来源

通过 RSSHub 聚合以下来源：

| 来源 | RSSHub 路由 | 说明 |
|------|------------|------|
| 机器之心 | /jiqizhixin/ | AI 专业媒体，论文解读 + 行业动态 |
| 量子位 | /qbitai/ | AI 热点新闻，更新快 |
| 36kr AI | /36kr/motif/452 | 综合科技媒体的 AI 频道 |

RSSHub 实例部署在 kada 服务器上，确保稳定性。

## 整体架构

```
RSSHub (自建实例)
    ↓ 定时抓取
kada 后端 (Express)
    ↓ 每 20 分钟
DeepSeek API → 筛选 + 总结
    ↓ 缓存
/api/news 接口
    ↓ 前端请求
聚合页首页 → 资讯模块渲染
```

## 后端设计（kada 服务器）

### 新增接口

**GET /api/news**

返回 20 条经过 AI 筛选和总结的资讯。

响应格式：
```json
{
  "updatedAt": "2026-05-04T10:00:00Z",
  "items": [
    {
      "id": "unique-id",
      "title": "GPT-5 发布：多模态能力全面升级",
      "summary": "OpenAI 发布 GPT-5，推理和多模态能力显著提升，支持实时视频输入。",
      "source": "机器之心",
      "category": "AI",
      "publishedAt": "2026-05-04T08:00:00Z",
      "url": "https://example.com/article"
    }
  ]
}
```

### 数据处理流程

1. **定时抓取：** 每 20 分钟从 RSSHub 拉取三个来源的最新资讯
2. **AI 筛选：** 将原始资讯列表发送给 DeepSeek，让 AI 从中挑选最有价值的 20 条
3. **AI 总结：** 对每条资讯生成 50 字以内的中文总结
4. **分类标签：** AI 自动判断分类（如 AI、硬件、应用、研究等）
5. **缓存：** 结果缓存在内存中，前端请求时直接返回

### DeepSeek 调用设计

筛选和总结在一次调用中完成，发送所有原始资讯给 DeepSeek，让它返回筛选后的 20 条及每条的总结和分类。

## 前端设计（聚合页）

### UI 一致性

资讯模块复用「构建」模块的布局模式：

- **左侧标题区：** sticky 定位，标题「资讯 / NEWS」，风格和「构建 / BUILDS」一致
- **右侧内容区：** 列表流排列
- **字体/颜色/间距：** 全部复用现有 CSS 变量
- **暗色/亮色主题：** 自动适配
- **移动端：** 左侧标题变顶部标题，卡片全宽

### 资讯卡片

每条资讯为一个横向卡片，包含：
- 左侧：分类标签（如 AI、硬件、应用）
- 中间：标题（加粗）+ AI 总结（灰色）
- 右侧：日期
- 点击展开：显示更多摘要 + 原文链接

### 交互

- 页面加载时请求 /api/news 接口
- 加载中显示骨架屏
- 请求失败显示友好的错误提示
- 点击卡片平滑展开/收起
